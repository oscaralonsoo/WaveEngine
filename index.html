Engine Overview

Building a solid foundation for future game development

Powerful C++ 3D Game Engine

Wave Engine is a custom 3D game engine developed in C++ using OpenGL as the main graphics API. It integrates several external libraries such as Assimp (for 3D model loading), DevIL (for texture management), and ImGui (for the user interface).

Version 2.0 has evolved beyond simple FBX dependency, implementing a complete resource management system with custom file formats and significant rendering optimizations.

The engine now features an advanced editor interface with scene serialization, resource management, and acceleration structures for optimal performance.

Engine Features

Powerful tools for game development and 3D visualization

Console

The console logs all engine events and processes, such as:

  • Loading geometry (via ASSIMP)
  • Loading textures (via DevIL)
  • Resource management operations
  • Initialization of external libraries
  • Application flow and error messages

Interactive options include:

  • Clear: Erases all current console messages
  • Log filters: Enable/disable display of specific log types (info, warnings, errors)

Configuration

The configuration window is divided into five tabs:

  • FPS: Displays current frame rate and performance data
  • Window: Customize application window size, resolution, fullscreen, borderless mode
  • Camera: Adjust sensitivities, reset settings, view position, change active camera
  • Renderer: Toggle face culling, wireframe mode, background color, debug visualizations
  • Hardware: View detailed system hardware information

Scene Management

Complete control over your 3D scene with:

  • Hierarchy: Full GameObject hierarchy with parent-child relationships
  • Inspector: Detailed object properties and transformation options
  • Scene Serialization: Save and load scenes to custom file formats
  • Runtime Transformations: Position, rotation, and scale at runtime
  • Mesh Data: View mesh statistics and toggle normal visualization
  • Material Editor: Preview textures with optional checker pattern

Resource Management

Advanced asset handling system:

  • Complete asset pipeline with custom file formats
  • Assets stored in "Assets" folder with optimized versions in "Library"
  • Reference counting for efficient resource loading
  • Automatic Library regeneration from Assets and metadata
  • Runtime asset importing

Performance Optimizations

  • Frustum Culling: Skip rendering objects outside camera view
  • Octree Spatial Partitioning: Accelerates rendering and object selection
  • Debug Visualizations: Toggle AABBs, octree nodes, and frustum
  • Optimized Rendering: Significant performance improvements

Camera System

  • Configurable camera component with adjustable parameters
  • Object selection using raycasting with octree
  • Visual feedback for selection operations
  • Active frustum culling for cameras
  • Multiple camera support

Object Properties

Provides detailed information and transformation options:

  • GameObject: Set active camera, reparenting, create/delete
  • Gizmo: Change mode (World/Local)
  • Transform: Modify position, rotation, scale
  • Mesh: Select from Assets window, normal visualization
  • Material: Select textures from Assets, preview options
  • Camera Component: Frustum culling, debug visualization

New Core Features in v2.0

Custom File Formats: Proprietary formats for models, textures and scenes with metadata files storing import settings and dependencies.

Import Settings: Basic implementation for different asset types:

  • Textures: Control filtering modes, max texture size, flip options
  • Meshes: Configure global scaling, axis configuration, post-processing options

Metadata: All import settings saved in .meta files to ensure proper regeneration of the Library folder.
